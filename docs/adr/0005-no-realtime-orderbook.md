# ADR-0005 — Отказ от real-time реконструкции стакана

**Статус:** принято · **Дата:** 2025

## Контекст

Заманчивая фича — «карта плотностей» / тепловая карта стакана (L2 order book).
Чтобы делать её честно в реальном времени, нужно подписаться на diff-стрим стакана,
поддерживать локальную копию книги, обрабатывать снапшоты+апдейты и хранить это.

## Решение

**Не делать** real-time реконструкцию стакана в MVP. Если и трогать тему — только как
**стретч** в виде периодических снапшотов (раз в N секунд) с биннингом по ценовым уровням.

## Почему

- L2-diff-стрим — большой трафик и нетривиальная логика синхронизации (легко
  рассинхронить книгу и показывать мусор).
- Хранение полной книги во времени — отдельная серьёзная задача по объёму данных.
- Аналитической ценности для скрининга это добавляет мало относительно стоимости;
  по приоритетам проекта (SQL + пайплайн) это ловушка scope creep (см. грабли §10).

## Альтернативы

- *Полная реконструкция книги* — отклонено: стоимость >> ценность для MVP.
- *Снапшоты + биннинг* — приемлемо как стретч, без обязательств.

## Последствия

- В MVP нет хитмапа стакана. Витрина строится на OHLCV/funding/OI — этого достаточно
  для заявленных метрик скрининга.
